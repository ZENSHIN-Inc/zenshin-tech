# 生成 AI ツール戦略

## 2026年7月版<br>**Claude と Codex** を抑えれば十分な理由

![w:240](../assets/brand/ZENSHIN-logo-white.webp)

---

<!-- _class: center -->

# 結論から

**2026年7月現在、追うべきは次の 2 つだけです。**

| 会社 | モデル | CLI ツール |
| --- | --- | --- |
| **Anthropic** | Claude | **Claude Code** |
| **OpenAI** | GPT-5.5 / 5.6 | **Codex** |

- この 2 つを使い込めば、世の中のほとんどの AI ツールは**代替できる**
- 新機能はこの 2 社が最初に実装し、他社は後追いになる

このスライドでは「**なぜそう言い切れるのか**」を順に説明します。

---

<!-- header: 1. 業界地図 — 基盤モデルは 3 社寡占 -->

# 強いモデルを作れるのは実質 3 社

| 会社 | モデル | 現在地 |
| --- | --- | --- |
| **Anthropic** | Claude | トップ争い |
| **OpenAI** | GPT-5.5 / 5.6 | トップ争い |
| **Google** | Gemini | 一歩遅れ |

- 中華系（**Kimi K2**、**DeepSeek** など）も台頭しているが、上記 3 社には届いていない
- **この 1 年で起きたのは 3 社内での順位の入れ替わりだけ**（画像・動画生成では各社の個性が出るが、勢力図は不変）

---

# 我々の業務の本質は「文書作成」

| 業務 | 正体 |
| --- | --- |
| メール・企画書・報告書 | **文書作成** |
| コード | **文書作成**（プログラムもテキスト） |
| スライド生成 | **文書作成**（構成と文章が本体） |
| グラフ・可視化 | 文書を **コードで可視化** したもの |

> 各社のバトルは **「どのモデルが文書とコードの生成に最も向いているか」** の勝負。派手なニュースに惑わされず **この 2 社** を見ておけばよい。

---

<!-- header: 2. 世の中の AI ツールの正体 -->

# ほとんどの AI ツールは「ラッパー」

Genspark、Manus などのツールの中身は、**Claude や GPT に適した指示文を被せているだけ**のケースがほとんど。

- モデルが賢くなるほど、ラッパー側の工夫の価値は**目減りする**
- 実際、この 1 年でラッパー系ツールの優位性はほぼ消えた

> **業務特化ツールの判断基準**: 中身がラッパーなら **スキルを設定した Claude Code / Codex でほぼ同じことができる**。買う前に「生の 2 社 + スキル設定」で代替できないかを試す。

---

# 特化ツールとの付き合い方 — V0 の例

**要所で手軽な特化ツールが刺さる場面はある。**

- 例: **V0**（ノーコードでシステムのプロトタイプを作るツール）
- 月 $20 程度なら、特定用途に課金する価値があるケースも

<div class="callout warn">

特化ツールは**すぐ新しいツールに追い抜かれ**、そこで培った使い方のノウハウが**陳腐化しやすい**。リスクを織り込んで課金額を決める。

</div>

<div class="callout good">

Claude と Codex は **1 年間トップランナーで走り抜いた実績**がある。学習投資を積み上げる先としては、この 2 つが最も安全。

</div>

---

<!-- header: 3. なぜこの 2 社が突出するのか -->

# 差がつくのは「ツールの使いこなし」

モデル単体の賢さ以上に、**外部ツールをどれだけうまく使えるか**で差がつく時代。

| 仕組み | できること |
| --- | --- |
| **スキル** | 業務の手順・知識を読み込ませて専用処理をさせる |
| **MCP・コネクター** | Gmail・カレンダー・Excel などと直接つながる |
| **プラグイン** | 用途ごとの機能拡張 |

Claude と OpenAI はこの「**ツールを使わせる仕組み**」の作り込みが頭一つ抜けており、Google は一歩遅れている。

---

# CLI 版こそが本命 — Claude Code と Codex

**CLI 版（Claude Code / Codex）にできること**

- ローカル PC のファイルに直接アクセスして作業する
- **PC 上のブラウザを操作**する
- **長時間ループで動き続ける**タスクをこなす
- スキル・MCP を接続して業務専用の処理を組む

<div class="callout warn">

Web ブラウザ版とは別物。**Web 版だけでは実力の半分も出ない。CLI 版の使いこなしが 2026 年の分岐点。**

</div>

---

# 2 社を追えば新機能は全部押さえられる

Claude と OpenAI は**新機能のリードカンパニー**。

- あらゆる新機能は、**この 2 社のどちらかが先に実装**する
- 片方が出せば、**1 ヶ月ほどでもう片方が追いかけて実装**する
- 両社は同じような概念でツール・機能を拡張している

<div class="callout info">

スマホでいう **iPhone と Pixel** の関係。この 2 つを出発点として押さえるのが、いま取れる最善の選択。

</div>

# まとめ

1. 強い基盤モデルは **3 社寡占**（Anthropic / OpenAI / Google）、
   世の中の AI ツールの大半は**そのラッパー**
2. 我々の業務の本質は**文書作成** — 勝負は文書・コード生成の 2 社
3. 差がつくのは**ツールの使いこなし**。スキル・MCP・ブラウザ操作・
   長時間実行まで扱える **CLI 版（Claude Code / Codex）が本命**
4. 新機能はこの 2 社が先行し、互いに約 1 ヶ月で追随 — **2 社だけ追えばいい**
5. 課金は **両方 $200 が推奨**、最低でも $200 + $20 の併用から

2026年7月現在、
**Claude Code と Codex をどっしり使い込むのが最善手**です。
