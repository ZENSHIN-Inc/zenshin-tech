| 会社 | モデル | CLI ツール |

# ほとんどの AI ツールは「ラッパー」

Genspark、Manus などのツールの中身は、**Claude や GPT に適した指示文を被せているだけ**のケースがほとんど。

- モデルが賢くなるほど、ラッパー側の工夫の価値は**目減りする**
- 実際、この 1 年でラッパー系ツールの優位性はほぼ消えた

> **業務特化ツールの判断基準**: 中身がラッパーなら **スキルを設定した Claude Code / Codex でほぼ同じことができる**。買う前に「生の 2 社 + スキル設定」で代替できないかを試す。

# 差がつくのは「ツールの使いこなし」

モデル単体の賢さ以上に、**外部ツールをどれだけうまく使えるか**で差がつく時代。

| 仕組み | できること |
| --- | --- |
| **スキル** | 業務の手順・知識を読み込ませて専用処理をさせる |
| **MCP・コネクター** | Gmail・カレンダー・Excel などと直接つながる |
| **プラグイン** | 用途ごとの機能拡張 |

Claude と OpenAI はこの「**ツールを使わせる仕組み**」の作り込みが頭一つ抜けており、Google は一歩遅れている。

---

# CLI 版こそが本命 — Claude Code と Codex

**CLI 版（Claude Code / Codex）にできること**

- ローカル PC のファイルに直接アクセスして作業する
- **PC 上のブラウザを操作**する
- **長時間ループで動き続ける**タスクをこなす
- スキル・MCP を接続して業務専用の処理を組む

<div class="callout warn">

Web ブラウザ版とは別物。**Web 版だけでは実力の半分も出ない。CLI 版の使いこなしが 2026 年の分岐点。**

</div>

---

# 「Web 版とは別物」の中身

| | Web 版（ブラウザ） | CLI 版（ローカル） |
| --- | --- | --- |
| ファイル | 添付した分だけ | **PC 内全部を直接編集** |
| 操作 | チャット回答まで | **アプリも自分の Chrome も操作** |
| 実行時間 | 1 問 1 答が基本 | **数時間の放置実行** |
| 積み上げ | 会話ごとにリセット | **スキル・設定が資産化** |

<div class="callout info">

「ローカル LLM」ではなく、**モデルはクラウド最上位のまま、手足だけがローカルに来る**。ブラウザは**ログイン認証ごと操作できる**。

</div>

---

# 2 社を追えば新機能は全部押さえられる

Claude と OpenAI は**新機能のリードカンパニー**。

- あらゆる新機能は、**この 2 社のどちらかが先に実装**する
- 片方が出せば、**1 ヶ月ほどでもう片方が追いかけて実装**する
- 両社は同じような概念でツール・機能を拡張している

<div class="callout info">

スマホでいう **iPhone と Pixel** の関係。この 2 つを出発点として押さえるのが、いま取れる最善の選択。

</div>

# 推奨する課金プラン

| 使い方 | 目安 |
| --- | --- |
| まず試す・ライトな通常業務 | $20 |
| コーディングや大量の資料作成が入る | $100 |
| コーディング多用のエンジニア | **$200** |

- おすすめは **2 社とも $200**。様子見なら **$200 + $20 併用**から
- 月 $200 ≒ **約 3 万円で GPT-5.5 / Opus 4.8 が使えるのは破格**
- **ミソはサブスク定額で最上位級が使えること**（従量課金ではない）

---

# 将来の値上げに備える

- 超トップモデル（**GPT-5.6 / Fable 5 級**）は、今後**従量課金・高額化**の可能性
- 一歩後ろを **GLM-5.2・Qwen・DeepSeek・Kimi・MiniMax** などの
  **コスパモデル**が追走 — **約 1 年で今のトップ水準に到達**する
- いま 2 社で貯めるノウハウは、
  **スキル化 + 省コンテキスト・省トークンの業務設計**

> このノウハウは**コスパモデルにそのまま移植できる**。乗り換える頃には追走勢も**実務ラインを超えてくる**ため、「コスパモデル + スキル」で業務を回す**保険になる**。

---

# この戦略が変わるタイミング

- 転機は、**2 社が定額サブスクをやめて主力モデルまで従量課金化**したとき
- そのときの選択は 2 つ — **腹をくくって従量課金**（AI 課金沼）か、
  **コスパモデルへの乗り換え**か
- どのモデルも**スキル・MCP・コネクターと同じ概念**で拡張されており、
  **積んだノウハウは乗り換え先でもそのまま効く**

> 2 社の進化は「よりオープン・外部接続・自律・長時間」の方向で、**特定モデルへのロックインは起きにくい**。それまでは 2 社で学び切り、その時が来たら **GLM 系などコスパモデルへ乗り換え**ればいい。

---

# 副次効果 — AI ニュースの解像度が上がる

2 社のツールを使い込むほど、業界の見え方が変わります。

- 新しい AI ツールのニュースを見たとき、
  「これは Claude の定期実行機能を使っているな」
  「これは既存機能のラッパーだな」と**裏側が推測できる**
- 「すごいものが突然出た！」という騒ぎにも**慌てずに済む**
- 一段高い視点から、俯瞰して AI ニュースを評価できるようになる

<div class="callout good">

ツールの習熟が、そのまま**情報の目利き力**になる。

</div>

# まとめ

1. モデルは **3 ティア構造**。トップは **Opus 4.8 / GPT-5.5** の 2 強
2. 世の AI ツールの大半は**そのラッパー**。業務の本質は**文書作成**
3. 差がつくのは**ツールの使いこなし**。スキル・MCP・ブラウザ操作・
   長時間実行まで扱える **CLI 版（Claude Code / Codex）が本命**
4. 新機能はこの 2 社が先行し、互いに約 1 ヶ月で追随 — **2 社だけ追えばいい**
5. 課金は用途別に **$20 / $100 / $200** — 推奨は **2 社とも $200**。
   **約 3 万円の定額で最上位級**が使えるのは破格
6. **定額が崩れたらコスパモデルへ乗り換え**。それまで 2 社で学び切る

2026年7月現在、
**Claude Code と Codex をどっしり使い込むのが最善手**です。
